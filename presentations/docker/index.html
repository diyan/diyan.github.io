Docker engine networking

  • docker0 virtual ethernet bridge is used by default
  • 172.17.42.1/16 CIDR by default (bye bye, Dev-Pro Wi-Fi)
  • --bip=192.168.100.1/24 in /etc/default/docker to override
  • --dns=, --dns-search= to setup DNS (container's /etc/resolv.conf)
  • --link= to link containers together (container's /etc/hosts)
  • --net=bridge is default, see docker0 above
  • --net=host to use real network from host
  • --net=container:NAME_or_ID to use network from other container
  • --net=none to manually configure network in container
  • -p port:port to let container accept incoming connections

Docker storage engines

Docker chooses a storage driver in this priority order

  • aufs is fast, - big files, no mainline Kernel but Ubuntu/Debian
  • overlay(fs) is fast, - big files, "proper" aufs, mainline Kernel 3.18
  • devicemapper is moderate for direct-lvm / slow for loop-lvm, + big files, developed by RedHat
  • btrfs is moderate, + big files, maybe good if you already have btrfs for something else
  • vfs is super slow, not compact (copy on write == just copy), implemented for tests
